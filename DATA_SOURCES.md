# Data Sources

This project currently uses data from three different places, depending on which screen you are looking at.

## 1. App Dashboard Data

The in-app dashboard at `/dashboard` reads directly from the PostgreSQL database through SQLAlchemy.

Main route:
- [app/routes/dashboard.py](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/app/routes/dashboard.py)

Main tables used there:
- `equipments`
- `equipment_metadata`
- `daily_inspections`
- `telemetry_records`
- `telemetry_alerts`
- `places`
- `users`
- `equipment_types`

This means the KPI cards, inspection counts, alert counts, telemetry counts, charts, and serviceability tables on the project dashboard are all database-backed.

## 2. Live Telemetry Cards

The live equipment cards on `/dashboard` are refreshed from these APIs:
- `/api/telemetry/dashboard`
- `/api/telemetry/alerts`

Relevant code:
- [app/routes/telemetry.py](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/app/routes/telemetry.py)
- [app/core/telemetry.py](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/app/core/telemetry.py)

Those APIs get data from:
- the in-memory `telemetry_hub` cache for recent live updates
- or the PostgreSQL tables `telemetry_records` and `telemetry_alerts` as fallback

So this section is still based on your local project data, not Tableau.

## 3. Demo / Seed Data

Most of the current data in the database was generated by the seed scripts and bootstrap helpers.

Relevant files:
- [app/core/bootstrap.py](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/app/core/bootstrap.py)
- [app/core/bulk_demo_data.py](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/app/core/bulk_demo_data.py)
- [seed_demo_data.py](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/seed_demo_data.py)

That seeded:
- users
- roles
- places
- equipment
- equipment metadata
- inspections
- workflow history
- telemetry history
- alerts

## 5. Embedded Tableau Dashboard Page

The new page at `/tableau-dashboard` does **not** query your local DB directly.

Relevant files:
- [app/routes/pages.py](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/app/routes/pages.py)
- [app/templates/tableau_dashboard.html](/Users/rashiajaygupta/Documents/GitHub/EMERGING-LABS/app/templates/tableau_dashboard.html)

That page only embeds this published Tableau Public workbook:
- `https://public.tableau.com/app/profile/rashi.gupta2443/viz/EmergingLab/Dashboard1`

Important distinction:
- `/dashboard` = live project data from PostgreSQL
- `/tableau-dashboard` = embedded external Tableau Public viz

If your Tableau Public workbook was built from the exported CSV files, then its underlying data originally came from this same project DB, but once published the app is only embedding the Tableau URL.

## Short Answer

You are getting data from:

1. PostgreSQL for the project dashboard and telemetry APIs
2. Seeded demo data inside that PostgreSQL database
3. CSV exports generated from that same database for Tableau
4. Tableau Public embed for the `/tableau-dashboard` page
